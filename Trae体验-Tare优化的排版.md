# Trae 概述

> TRAE 深度融合 AI 能力，是一名能够理解需求、调用工具并独立完成各类开发任务的“AI 开发工程师”，帮助你高效推进每一个项目

## 默认编辑模式

### 使用了 Trae 的智能体的功能

> [自定义智能体](https://docs.trae.cn/ide/custom-agents-ready-for-one-click-import)

#### Trae 智能体 - UI 设计师

- 我导入了 UI 设计师
- 使用项目 teach-web，架构是相对老一些的**nuxt2**
- 设计实现一个新的页面，在 teach-web（爱投顾 web 端）项目中实现一个新的页面
- UI 设计是，金融界风格的页面
- Trae 使用的模型是 Doubao-Seed-Code
- 整体来说，响应速度中等，实现的页面，与 UI 相去较远，也没有符合 teach-web 项目本身的 UI 风格
- 完成后，首次运行项目，无法启动（实现完成后，没有主动复查一下页面，解决报红的问题）
- 通过后续多次对话解决了问题

> **总结**
>
> 1. 默认的 ide 模式，是现在 AI 辅助编辑都在使用的 plan 模式（即会先规划好流程，在按流程逐步实现）
> 2. 使用顺手程度上，比较符合以往的开发体验
> 3. 主要是 Trae，基于开源的 Visual Code 开发的，而且可以直接导入 Visual Code 和 Cursor 的已有插件和配置（快捷键、代码片段、插件等）
> 4. 模型能力上，与我目前正在使用 claude 4.5 相比，还是差一些
> 5. claude4.5 使用了相同提示词，实现的 UI 和设计图相近，还参考了 teach-web 项目本身的风格
> 6. 观察其思考过程，发现因为智能体的缘故，流程还是不错的
> 7. Trae 的模型 Doubao-Seed-Code 实现的页面不太行的原因，可能是模型能力本身差一些或者缺乏训练

#### Trae 智能体 - 前端架构师

- 以 jrj-ai-assistant 项目为例，使用的智能体前端架构师
- jrj-ai-assistant 是新的技术架构，monorepo，项目本身是 vue3 的，使用的是 typescript 的 setup 模式
- 项目本身集合了所有 AI 助手的 7 个子应用
- 提问 1：首先分析一下，当前这个项目，尽可能的去了解项目的内容
- 提问 2：当前项目，有哪些优点？哪些缺点？
- 提问 3：针对改进建议的 1，先规划一下，改进的步骤（下方是建议 1）

> - ### 组件库优化
> - 重构组件库，统一组件设计规范
> - 合并功能重叠的组件(如 NullData 和 NullDataH5)
> - 完善组件文档和示例

- 提题 1 给出的结论，中规中矩，还有一些根据命名发散的结论，虽然不对，但联想能力还是正常的
- 提题 2 给出的结果，其他都还中规中矩，缺点中，提到了一个【性能优化空间】，这个姣好，指出的点也是正确的
- 提题 3，问题的本意是先给出步骤，确认后再开始改进代码，但是列出步骤后，没有让我确认，直接开始的优化
- 提题 3 的优化结果，整体来说达标，从开发者的角度来说，还是不太理想

- 优化过程中，给出了优化步骤（创建优化步骤的 Markdown 文件），但是只优化了文档中小部分
- 通过提问让其继续完成未完成工作时，此时再完成了所有组件的优化
- 但是，所有的组件的优化中，存在 3 个问题
- 问题 1 是，其中有个组件有报红的 warning，模型也发现了这个问题，这个报红很好解决，改一下写法就可以解决，但是模型翻来覆去检查、修改、又还原，最终还是没有解决，然后悄摸的放弃了，然后告诉我解决了
- 问题 2 是，其中有个组件，前面提到了要对其进行优化，但是看模型思考的过程，模型复查了 2 遍是否还有未优化的组件，都没有优化这个组件，而与这个组件类似的组件正常被优化了
- 问题 3 是，优化进程走到后面后，开始修改组件的部分尺寸了，这一点我认为是不好的

> **总结**
>
> 1. 整体过程，还是可以的，还是有一些我认为不太行点在上面有列出
> 2. 整个过程,持续了将近 30 分钟，用时较长，中间 2 次到达了思考次数极限
> 3. 优化的内容，较为简单
> 4. 对于企业的私有项目来说，增加了较多无用的内容，譬如十分详尽的组件文档和类型文件，当然，这一点可以在提问时进行约束
> 5. 长期使用的话，需要总结一套 针对不同项目，不同类型需求的提示词模板，有针对的使用 AI
